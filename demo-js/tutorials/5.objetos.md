# Objetos

## Objetos literales

Los objetos son colecciones de datos relacionados junto con las funcionalidades necesarias para esos datos. Para ello se agrupan una serie de propiedades que puedes corresponder a cualquiera de los tipos que ya conocemos, incluyendo otros objetos. Cuando a una propiedad se le asigna una funci√≥n se denomina m√©todo, y confiere al objeto las funcionalidades antes mencionadas.

Lo que diferencia a los objetos de cualquier otro tipo de dato es su **mutabilidad**: se pueden a√±adir, eliminar o modificar propiedades, es decir cambiar o mutar el objeto, sin necesidad de reasignarle un nuevo valor, por lo que no es un impedimento que haya sido declarado mediante const.

La forma m√°s sencilla de crear un objeto se conoce como **objetos literales** y permite indicar entre llaves las propiedades del objeto con sus correspondientes valores despu√©s de dos puntos, separando mediante comas las sucesivas propiedades.

```js
// Declaramos una variable y le asignamos como valor un objeto
const person = {
  name: 'Pepe',
  age: 33,
};
console.log(person, typeof person); // { name: 'Pepe', age: 33 } object
```

El operador punto (.) permite acceder al conjunto de propiedades del objeto, para a√±adir, modificar o mostrar por consola cualquier propiedad,

```js

  name: 'Pepe',
  age: 33,
};
// A√±adimos nuevas propiedades
// En este caso con un valor de tipo objeto
person.address = { street: 'c/ Pez', number: 12, city: 'Madrid' };
// Modificamos el valor de una propiedad
person.age = 34;
// Mostramos por consola el objeto
// con los cambios anteriores
console.log(person); // {name: 'Pepe', age: 34,
// address: { street: 'c/ Pez', number: 12, city: 'Madrid' }}
// Accedemos a una sola propiedad del objeto
// y en este caso la mostramos por consola una sola
console.log(person.name); // Pepe
```

---

ü•∑ El operador delete permite eliminar completamente una propiedad, aunque no es una pr√°ctica muy habitual

## Arrays

Un tipo particular de objetos que utilizaras en tus pr√≥ximos ejercicios son los **arrays**. En este caso las propiedades no tienen nombre sino que se numeran sucesivamente desde O.

---

ü•∑ En JS es importante tener en cuenta que, en definitiva son **objetos**, con la misma flexibilidad que los dem√°s objetos. Su n√∫mero de elementos puede aumentar o disminuir en cualquier momento, al contrario de lo que sucede con los arrays estrictos habituales en muchos lenguajes de programaci√≥n.

---

La creaci√≥n de los arrays es similar a la de los objetos, pero en este caso utilizando los **corchetes** ([]). La variable correspondiente se declara con const y recibe un nombre que indique su car√°cter de lista de m√∫ltiples elementos, mediante un plural (users) o una referencia a diversos incontables (people).

Los elementos o **items** del array pueden ser de cualquier tipo, y de diferentes tipos unos y otros, aunque igual que dec√≠amos respecto a los cambios de tipos en las variables, ser√≠a una buena pr√°ctica mantener siempre un **√∫nico tipo** invariable en todos los elementos de un array

```js
// Declaramos mediante const una variable
// y le asignamos un array de strings
const users = ['Pepe', 'Luisa'];
console.log(users, typeof users); // [ 'Pepe', 'Luisa' ] object
```

Mediante los corchetes tambi√©n podemos acceder a cualquier posici√≥n de un array, indicando su √≠ndice, para modificar su valor o utilizarlo, por ejemplo mostr√°ndolo en la consola.

Como objetos que son, el operador . permite acceder a sus propiedades, en este caso comunes a todos los arrays, que proporcionar una serie de m√©todos junto con la propiedad length, correspondiente a la longitud del array.

```js

// La propiedad length indica siempre la longitud del array
console.log(users.length); // 4
// Podemos acceder a un item indicando su √≠ndice o posici√≥n
console.log(users[0]); // Pepe
// Accediendo a un item por su √≠ndice, podemos modificar su valor

console.log(users[0]); // Jose
// A partir de la longitud del array
// Podemos a√±adir nuevos elementos
// que tendr√°n como indice el siguiente al √∫ltimo

console.log(users); // [ 'Jose', 'Luisa', 'Elena', 'Ernesto', 'Sofia' ]
console.log(users.length); // 5
```

Como has visto, al conocer la longitud de un array, es f√°cil a√±adir un nuevo elemento en la posici√≥n siguiente a la √∫ltima. Tambi√©n podemos hacer eso mismo con alguno de los muchos metodos de los que disponen todos los arrays.

### M√©todos mutables de arrays

Los dos primeros m√©todos modifican el **final** del array

- array.push(value): **Agrega** uno o m√°s elementos al final de un array y devuelve la nueva longitud del array.

```js

const newLength = users.push('Sofia');
console.log(users); // [ 'Jose', 'Luisa', 'Elena', 'Ernesto', 'Sofia' ]
console.log(newLength); // 5
```

- array.pop(): **Elimina** el √∫ltimo elemento de un array y lo devuelve.

```js
const users = ['Pepe', 'Luisa', 'Elena', 'Ernesto'];
const deletedUser = users.pop();
console.log(users); // [ 'Pepe', 'Luisa', 'Elena' ]
console.log(deletedUser); // Ernesto
```

Tenemos tambi√©n otros dos que modifican el **principio** del array

- array.unshift(value): **Agrega** uno o m√°s elementos al inicio del array, y devuelve la nueva longitud del array.

```js
const users = ['Pepe', 'Luisa', 'Elena', 'Ernesto'];
const newLength = users.unshift('Sofia');
console.log(users); // [ 'Sofia', 'Pepe', 'Luisa', 'Elena', 'Ernesto' ]
console.log(newLength); // 5
```

- array.shift(): **Elimina** el √∫ltimo elemento de un array y lo devuelve.

```js
const users = ['Pepe', 'Luisa', 'Elena', 'Ernesto'];
const deletedUser = users.shift();
console.log(users); // [ 'Luisa', 'Elena', 'Ernesto' ]
console.log(deletedUser); // Pepe
```

Finalmente podemos modificar unicamente el order de nuestro array

- array.sort(): ordena el array de lo invoca. Si no recibe par√°metros, el orden ser√° el alfab√©tico ascendiente. Para otras ordenaciones, el par√°metro ser√° una funci√≥n responsable de establecer el orden.

```js

users.sort();
console.log(users); // [ 'Elena', 'Ernesto', 'Luisa', 'Pepe' ]
```

### M√©todos no mutables de arrays

En este apartado hay m√©todos muy potentes aunque algo complejos, que estudiaras m√°s adelante (forEach, map, filter, reduce, find, some, every).

Tambi√©n hay algunos muy sencillos de entender y utilizar. Como ves, al final mostramos por consola el array y puedes comprobar que no ha sido modificado.

- array.includes(): Determina si un array incluye un determinado elemento y retorna un booleano seg√∫n corresponda.

```js

let isIncluded = users.includes('Elena');
console.log(isIncluded); // true
isIncluded = users.includes('Sofia');
console.log(isIncluded); // false
console.log(users); // [ 'Pepe', 'Luisa', 'Elena', 'Ernesto' ]
```

- array.join(): devuelve una cadena en la que une todos los elementos del array. Podemos pasarle como par√°metro el car√°cter de separaci√≥n que debe agregar entre los elementos, aunque por defecto utilizar√° una coma (sin espacios).

```js

const listUsers = users.join(', ');
console.log(listUsers); // Pepe, Luisa, Elena, Ernesto
console.log(users); // [ 'Pepe', 'Luisa', 'Elena', 'Ernesto' ]
```

### Arrays multi dimensionales. Bucles anidados

Cuando los elementos de un array son arrays, tenemos un array de dos **dimensiones**. Si sucede lo mismo en el array interior sucesivamente, podemos tener n dimensiones. En algunas ocasiones encontraras el nombre de matrices para referirse a estas estructuras pero en JS no son otra cosa que arrays.

Para recorrer estos arrays necesitamos anidar un bucle for por cada dimensi√≥n

Veamos un ejemplo con dos dimensiones.

```js
const matrix = [
  [1, 2, 3],
  [10, 20, 30],
  [100, 200, 300],
];
for (let i = 0; i < matrix.length; i++) {
  const line = matrix[i];
  for (let j = 0; j < line.length; j++) {
    const item = line[j];
    console.log(`En la l√≠nea ${i + 1}, el valor ${j + 1} es  ${item} `);
  }
}
```
